# Developer Guide & API Reference 🛠️

This guide provides a comprehensive reference of all classes, methods, and cryptographic features within the **Integrity SDK**, detailing error codes, verification structures, and how to create custom plugins.

---

## 1. Class Reference: `IntegrityClient`

The `IntegrityClient` manages the asynchronous batching queue, hardware attestation, and signature loops.

### Constructor
```python
client = IntegrityClient(
    agent_id="my_agent",
    oracle_url="http://localhost:3001/ingest",
    batch_size_limit=50,
    flush_interval_sec=5.0,
    did=None,
    subagent_id=None,
    enable_full_recording=False
)
```
- `agent_id` (str): Unique identifier for the agentic harness.
- `oracle_url` (str): The ingest endpoint of the `integrity-oracle` server.
- `batch_size_limit` (int): Number of telemetry events before trigger flush.
- `flush_interval_sec` (float): Maximum seconds before flushing a partial batch.
- `did` (Optional[str]): Specify custom DID document string (autogenerated if None).

### Method: `log_telemetry`
Adds an event payload to the queue. Returns immediately.
```python
client.log_telemetry(
    metadata={"action": "trade", "price": 105.2},
    entropy=0.12,
    grounding=0.98,
    subagent_id=None
)
```

### Method: `log_model_switch`
Logs a model switch event. Can be triggered manually or automatically inside `log_telemetry`.
```python
client.log_model_switch(
    from_model="gpt-4o",
    to_model="llama-3",
    from_provider="openai",
    to_provider="together",
    reason="cost_optimization"
)
```

### Method: `log_hitl_action`
Logs a human-in-the-loop approval, rejection, or override action. Automatically calculates Levenshtein edit distance if `proposed_content` and `final_content` are provided.
```python
client.log_hitl_action(
    action_type="override",
    proposed_content="Buy 10 AAPL",
    final_content="Buy 15 AAPL",
    reviewer_did="did:xibalba:human_operator_42",
    review_latency_ms=1250.0,
    justification="Liquidity depth suggests higher trade size"
)
```

### Method: `spawn_subagent`
Spawns a child/sub-agent client inheriting the connection, batching, and directory configurations, but using a hierarchical sub-agent DID path (`parent_id.subagent_id`).
```python
sub_client = client.spawn_subagent("ScreenerSubagent")
```
* **Parameters**:
  - `subagent_id` (str): Unique suffix identifier for the child agent.

### Method: `shutdown`
Gracefully stops the background threads, flushes remaining telemetry, and stops the host telemetry sampler.
```python
client.shutdown()
```

## 2. Configuration & Environment Override

To run or initialize the SDK dynamically without hardcoding configurations inside code, use these environment variables:

| Environment Variable | Description |
| :--- | :--- |
| `INTEGRITY_AGENT_ID` | Override default runtime agent identifier. |
| `INTEGRITY_ORACLE_URL` | Set the target HTTP ingest API of the oracle. |
| `INTEGRITY_OTLP_ENDPOINT` | Set the OpenTelemetry OTLP/gRPC endpoint (default: `localhost:4317`). |
| `INTEGRITY_STORAGE_DIR` | Custom root directory for DID document generation. |

## 3. High-Fidelity Measurement Apparatus (v2.0)

### OpenTelemetry Integration
The SDK v2.0 is built on `opentelemetry-python`. On initialization, it automatically configures:
- **TracerProvider**: Captures inference spans and MCP tool call boundaries.
- **MeterProvider**: Emits host telemetry and cognitive quality metrics.
- **Batching**: Spans and metrics are batched and exported asynchronously via OTLP/gRPC.

### Host Telemetry Sampler
The macroscopic host observer samples system state every 15 seconds:
- **Metrics emitted**: `integrity.host.storage_flux.rw_ratio`, `integrity.host.storage_flux.path_entropy`, `integrity.host.network.ip_entropy`.
- **Requirements**: The process must have permissions to read `/proc` (on Linux) or use `psutil` hooks.

## 4. Drop-In Wrapper Reference: `IntegrityOpenAI`

A drop-in subclass replacing the standard OpenAI client to capture cognitive metadata seamlessly.

```python
from integrity_sdk import IntegrityOpenAI

client = IntegrityOpenAI(
    api_key="sk-...",
    agent_id="agent_quant_trader_01",
    oracle_url="http://localhost:3001/ingest"
)
```

- **Features**:
  - Automatically hooks streaming chunk deltas and reconstructs the full output.
  - Automatically calculates Type-Token Ratio (TTR) for token diversity.
  - Monitors and extracts prompt length, completion length, and latency parameters.

## 3. SQLite Offline Cache & HMAC Security Protocol

If the target `integrity-oracle` becomes unreachable, the client redirects all queued telemetry into the local SQLite WAL to ensure **zero data loss**:

- **Database Path**: `~/.integrity/offline_moat.db`
- **Crypto Protection (HMAC-SHA256)**:
  Every row written to SQLite is bound to a cryptographic HMAC signature generated using the secret seed derived from the agent's private DID key:
  $$\text{integrity\_hash} = \text{HMAC-SHA256}(\text{payload\_str}, \text{private\_key})$$
- **Verification**:
  Upon synchronization, the background worker re-evaluates the hash of each row. If any modification is detected (indicating offline database file tampering by an intruder), the SDK automatically **discards the row** and logs a security warning.

## 4. Concurrency Collision Prevention (Process Noncing)

To allow horizontal container scaling and multi-process execution of identical agent profiles, the batch-level anti-replay nonces are derived using the host process ID:

$$\text{nonce} = (\text{timestamp\_ms} \times 10000) + (\text{process\_id} \pmod{10000})$$

This bounds uniqueness to the process boundary, preventing Redis ingestion conflict lockouts.
